# Security Policy

## Supported versions

The latest published `0.1.x` release line receives security fixes on a best-effort basis.

## Reporting a vulnerability

Please do not open a public GitHub issue for a potential vulnerability.

Instead:

1. email the maintainer or use the private reporting channel available on GitHub
2. include the affected version, reproduction steps, and impact
3. allow reasonable time for triage before public disclosure

When possible, include:

- whether the issue affects the CLI, MCP server, or docs/examples
- whether a malicious prompt, config file, or environment variable is required
- any logs or proof-of-concept snippets
